# Data Quality & Reconciliation Checks

This project demonstrates validation rules and reconciliation queries used to monitor data quality, detect gaps, and improve trust in reporting outputs.

## Business Use Case
Analytics environments need strong controls to ensure data integrity across systems and reporting layers. This project includes examples of volume checks, duplicate detection, and cross-table reconciliation.

## Files
- `duplicate_record_check.sql`
- `null_critical_fields_check.sql`
- `cross_system_totals_check.sql`
- `source_to_target_count_check.sql`

## What This Demonstrates
- data quality monitoring
- reporting reliability controls
- reconciliation design
- operational governance
